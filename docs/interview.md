# 面试表达

## 一分钟项目介绍

ResearchFlow 是一个面向论文和技术资料的可溯源研究 Agent。用户上传 PDF 后，系统按页解析并切分文本，通过 Embedding 写入 Qdrant。提问时，模型利用 Function Calling 选择论文检索、联网搜索或实验指标 SQL 工具，LangGraph 负责工作流编排，最后由引用检查节点确保返回的论文引用都能映射到真实页码和 chunk。FastAPI 提供 API，PostgreSQL保存会话与工具轨迹，MCP 让外部 Agent 复用同一组研究能力，整个项目可以用 Docker Compose 部署。

## 高频追问

**为什么不是普通聊天机器人？**

普通模型只生成文本；ResearchFlow 能访问私有论文、结构化指标和联网来源，并保存每次工具执行和证据来源。

**RAG 如何减少幻觉？**

RAG 不保证模型绝不幻觉，但它缩小回答依据。项目进一步由程序生成 citation 对象并校验 chunk ID，避免模型自行编造页码。

**为什么选择 LangGraph？**

研究问题可能同时需要论文、网页和 SQL。显式 State、Node、Edge 让路由、失败降级和轨迹可观察，比隐藏在单个 prompt 里的循环更容易调试。

**如何防止 Text-to-SQL 破坏数据库？**

只开放实验指标表；先用 sqlglot 解析 AST，拒绝多语句、写操作、子查询、未授权表字段和函数，再强制 50 行限制，并在 PostgreSQL 事务中设置只读。

**为什么 MCP 不直接复制一套逻辑？**

MCP 与 REST API 都调用同一个应用服务层，协议只是入口。这样检索策略、安全策略和数据模型只有一份实现。

**当前最大限制是什么？**

第一版仅处理文本型 PDF，摄取任务运行在 API 进程中，适合单用户作品集。生产扩展应加入 OCR、任务队列、用户隔离、对象存储和可观测性。
